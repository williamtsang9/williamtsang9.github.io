    What I've done

    Here's a final project that I took part in building during my time at DBC. Many more projects to come.

  • Hummingbird

    • Summary


    Hummingbird was created with a purpose of scheduling text messages so that they could be sent at the right time. Our idea was to have the backend API always up and running so that frontend users could connect to it. The best way to envision this idea was through decoupled architecture. By decoupling the web app, the backend could actively batch up messages to be sent while the frontend users access the application to schedule their messages. The tech stack includes Ruby on Rails using the Twilio API for the backend and Ionic(AngularJS) framework for the frontend.


  • Tech Stack


  • Decoupled Archiecture
  • Frontend: Ionic Framework with AngularJS
  • Backend: Rails with PostgreSQL
