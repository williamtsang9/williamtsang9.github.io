Object Oriented Programming:
Inheritance vs Composition

11/16/14


What is Inheritance?

Let's start with the the word inherit. What does it mean? In a transitive way, inherit means to receive something from someone. Think of your facial features.. You've probably heard something like, "Oh you look more like your mom or dad." For me, people always say that I've inherited my mom's side of the genes because I have similar facial features to the males there. Class inheritance in Ruby is just the same! Think of Class inheritance as a father to son relationship. The son takes after his father with all of his traits, features and personality. (Maybe not the last one.. but take a look at the example below)

Right off the bat, you'll see a Class being created with a method called traits inside. This traits method will puts a string out that will define the Dad. Next, I create the Class Son and you'll notice that there is a < Dad next to it. You probably guessed right. That's the inheritance symbol. When you have this symbol, it means I'm going to inherit all the information, methods, variables within the targeted Class (in this case, Dad). Next, we instantiate these two classes by declaring objects with Dad.new and Son.new. Here's the trick, we call the method traits found in Class Dad on BOTH objects. Tell me what you see!

Although the Class Son is not quite Superman yet, he'll grow to be just like his Class Dad.

What is Composition?

Inheritance will only be a 1 to 1 copy from class to subclass. Eventually, as the program grows bigger, inherited and parent classes will slowly start to diverge to satisfy the functions written by the programmer. Composition is similar to inheritance but still different at the same time. They both share the idea of using methods and functions from one class to another but the way they do it is different. Here's where Composition shines. There are 2 ways that can "mimick" the uses of inheritance.
1. Share the parent class to a subclass by adding this line, (parentclasshere).new(self).(parentmethodhere)
This line calls the Parent class within the subclass and uses a method from the parent class. Awesome right?
2. Another method is to use modules and mixins. Just add the line module (modulenamehere). Also, use the line, include (modulenamehere) wherever you want to access the class of choice. Pretty interesting if you ask me.

Personally, I would start getting used to composition as I feel that it may come in handy on bigger projects.
